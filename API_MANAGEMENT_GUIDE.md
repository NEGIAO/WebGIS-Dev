# API 管理后台使用指南

## 概述

API 管理后台是一个专为管理员设计的数据分析和监控工具，用于查看和管理用户的 API 调用情况。

## 功能模块

### 1. 📊 用户统计
查看所有用户的 API 调用情况（最近 7/30/90 天）：
- **用户名**：用户账号
- **角色**：游客/注册用户/管理员
- **调用次数**：该用户的 API 调用总次数
- **活跃天数**：有调用的天数
- **平均响应时间**：所有调用的平均响应时间
- **最后调用**：最近一次调用的时间

### 2. 🔗 API 端点统计
查看各个 API 端点的调用情况：
- **API 端点**：被调用的 API 地址
- **调用次数**：该端点的总调用数
- **成功/失败**：成功和失败的调用次数
- **成功率**：成功调用的百分比
- **响应时间**：平均、最大、最小响应时间

### 3. 📝 调用日志
实时查看 API 调用日志（支持分页、过滤）：
- **时间戳**：调用时间
- **用户/角色**：调用者信息
- **API 端点**：被调用的 API
- **状态码**：HTTP 状态码（颜色标记）
- **响应时间**：本次调用的响应时间

**过滤选项**：
- 按用户名过滤
- 按 API 端点过滤
- 按时间范围过滤（7/30/90 天）

### 4. ⚙️ 配额配置
查看当前的 API 配额设置：
- **游客配额**：游客每日可调用的 API 次数
- **注册用户配额**：注册用户每日配额
- **管理员配额**：管理员配额（通常无限）

## 使用方式

### 访问管理后台

1. **首先以管理员身份登录**
   - 用户名：`admin`
   - 密码：后端 `SUPER_USER` 环境变量（默认 `123456`）

2. **进入管理后台**
   - 在应用菜单中找到"API 管理"或访问路由 `/admin/api-management`

### 查看用户统计

1. 进入"📊 用户统计"标签
2. 选择统计时间范围（7/30/90 天）
3. 点击"刷新"按钮查看最新数据
4. 点击表格行可以查看该用户的详细信息

### 分析 API 性能

1. 进入"🔗 API 端点统计"标签
2. 查看各端点的调用频率和性能指标
3. 识别性能问题（响应时间过长、错误率高的端点）

### 查看实时日志

1. 进入"📝 调用日志"标签
2. 使用搜索框按用户名或端点过滤
3. 使用分页导航浏览历史记录
4. 点击"刷新"按钮查看最新日志

### 查看配额配置

1. 进入"⚙️ 配额配置"标签
2. 查看各角色的每日 API 限额
3. 如需修改，编辑环境变量：
   ```bash
   GUEST_DAILY_API_QUOTA=50          # 游客每日限额
   REGISTERED_DAILY_API_QUOTA=500    # 注册用户每日限额
   # 管理员不限额
   ```
4. 重启后端服务使配置生效

## 后端 API 端点

### 获取用户统计
```
GET /api/admin/api-management/usage/by-user
参数:
  - days: 7|30|90 (统计天数)
  - limit: 1-500 (返回用户数)
```

### 获取端点统计
```
GET /api/admin/api-management/usage/by-endpoint
参数:
  - days: 7|30|90
  - limit: 1-200
```

### 获取调用日志
```
GET /api/admin/api-management/logs
参数:
  - limit: 1-1000
  - offset: 0+
  - username: 用户名过滤(可选)
  - endpoint: 端点过滤(可选)
  - days: 7|30|90
```

### 获取配额配置
```
GET /api/admin/api-management/quota-config
```

### 获取用户今日使用
```
GET /api/admin/api-management/user/{username}/today-usage
```

## 常见问题

### Q: 日志中没有显示数据？
A: 首次运行时，数据库表会自动创建。请确保：
1. 已有用户调用过 API
2. 查询时间范围包含调用的时间
3. 检查后端日志是否有错误

### Q: 如何导出统计数据？
A: 目前暂不支持直接导出。可以：
1. 使用浏览器开发者工具复制表格数据
2. 使用浏览器插件将表格导出为 CSV/Excel
3. 直接查询后端 SQLite 数据库

### Q: 配额为什么显示"无限制"？
A: 管理员账号没有每日限额限制。要设置限额，需要修改代码中的配额配置。

### Q: 如何清空调用日志？
A: 需要直接操作数据库：
```sql
DELETE FROM api_call_logs;
VACUUM;  -- 释放空间
```

## 数据安全提示

⚠️ 重要注意：
- 只有管理员可以访问 API 管理后台
- 日志包含用户调用记录，请保护好管理员账号
- 定期备份数据库以防数据丢失
- 不要在日志中记录敏感信息（如密钥等）

## 性能优化建议

1. **定期清理旧日志**
   - 调用日志表会不断增长
   - 建议每月清理 90 天前的日志

2. **添加索引**
   - 已为常用字段添加索引
   - 若查询速度慢，可添加更多索引

3. **考虑日志归档**
   - 对于长期运行的系统
   - 考虑将旧日志导出到其他存储
