## 🎯 修改原因

### 问题分析

**核心症状**: 用户反映下载任务总是失败

**链路追踪**:
1. 前端提交下载请求 ✅
2. 后端创建任务 ✅
3. **后端处理瓦片下载** ❌ → 返回 None，导致处理中断
4. 前端轮询任务状态 → 收到无效响应
5. 下载失败

**根本原因识别**:
- 后端坐标转换函数返回 None (缺少 return 值)
- 后端过期时间计算返回 None (缺少 return 值)
- 前端对无效响应处理不当（无防御性编程）
- 请求参数验证不充分（允许极端值通过）

## 🔍 技术细节

### 坐标转换验证

**EPSG:3857 → EPSG:4326 转换公式**:

$$\text{lon} = \frac{x}{20037508.342789244} \times 180$$

$$\text{lat} = \frac{180}{\pi} \times \left(2 \arctan\left(e^{\frac{y}{20037508.342789244} \times \pi}\right) - \frac{\pi}{2}\right)$$

其中：
- `x, y`: Web Mercator 投影坐标（米）
- `20037508.342789244`: Web Mercator 范围常数
- `lon, lat`: WGS84 坐标（度）

### 分辨率与缩放级别的关系

$$\text{zoom} = \log_2\left(\frac{\text{meters\_per\_pixel}}{\text{resolution\_m}}\right)$$

其中：
- `meters_per_pixel = INITIAL_RESOLUTION / 2^zoom \times \cos(\text{latitude})`
- `INITIAL_RESOLUTION = 156543.03392804097` (赤道处像素分辨率)
