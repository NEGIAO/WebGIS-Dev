# 游客配额权限与结构化认证错误

## 📅 日期和时间
2026-06-11 10:30

## 📝 修改内容

### 问题分析

**核心症状：** 游客身份请求天气、搜索、AI 等后端 API 时直接被 401 拒绝，而非走配额制度。

**根本原因链：**
```
前端无 token 请求 → 后端 require_login 无 share-mode → 直接 401 拒绝
→ 前端收到 401 → clearAuthSession() → 跳转登录页
→ 用户看到"请重新登录"而非"游客权限不足，请注册"
```

**受影响模块：** 认证依赖注入、外部代理、Agent Chat、前端 HTTP 客户端

### 修复方案

#### 1. 后端 `dependencies.py` — 结构化错误 + 游客配额依赖

- 401 响应改为结构化 detail：`{code: "GUEST_NO_TOKEN", message: "..."}`
  - `GUEST_NO_TOKEN`：无 token 且无 share-mode
  - `SESSION_EXPIRED`：会话过期
  - `SESSION_ERROR`：会话查询异常
- 新增 `require_api_access_or_guest` 依赖：无 token 时自动创建临时 guest session 并走配额检查，而非直接 401

#### 2. 后端 `routes.py` / `external_proxy.py` / `statistics.py` — 端点依赖替换

- Agent Chat 端点（config / completions / default-proxy / user-config / models / preference / proxy）从 `require_login` 改为 `require_api_access_or_guest`
- `external_proxy.py`（天气、搜索、地理编码等 9 个端点）从 `require_api_access` 改为 `require_api_access_or_guest`
- `statistics.py` 从 `require_api_access` 改为 `require_api_access_or_guest`
- `update_agent_user_config` 保留 `require_login`（写操作需身份验证）

#### 3. 前端 `client.js` — 401 差异化处理

- 解析 `detail.code` 字段区分错误类型
- `GUEST_NO_TOKEN`：仅 warning 提示，不清除会话，不跳转登录
- `SESSION_EXPIRED` / `SESSION_ERROR`：清除会话 + error 提示（原逻辑）
- `handleApiError` 新增 `isGuestInsufficient` 分支

### 测试方案

1. 无 token 请求天气 API → 应返回正常数据（走 guest 配额）
2. 无 token 请求 AI Chat → 应返回正常数据（走 guest 配额）
3. guest 配额用完 → 应返回 429 + "今日额度已用完"
4. 会话过期请求 → 应返回 401 + 清除会话 + 跳转登录
5. 已登录用户请求 → 正常走用户配额
