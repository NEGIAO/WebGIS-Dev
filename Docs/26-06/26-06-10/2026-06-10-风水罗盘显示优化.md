# 风水罗盘显示优化

## 📅 日期和时间
2026-06-10 14:30 / 2026-06-10 16:00 更新

## 📝 修改内容

### 1. 半径判断逻辑优化
- **问题**：原 `isCompassVisibleInView` 使用简单的圆与矩形相交判断，导致罗盘在视图边缘附近容易被裁切
- **解决**：计算罗盘圆心到视图矩形的最近距离，当该距离 <= 罗盘半径 × 1.5 时认为可见
- **逻辑**：扩展检测范围，确保罗盘在接近视图边缘时仍能正常显示

### 2. 缩放级别无限制
- **问题**：原使用固定的 `minResolution` 或 `maxResolution` 控制显隐，缩放地图时罗盘自动隐藏
- **解决**：移除图层级别的缩放限制，罗盘在任意缩放级别下都可显示
- **逻辑**：不在图层上设置 `setMaxResolution` / `setMinResolution`，由用户自行控制显隐

### 3. 罗盘渐变背景增强
- **问题**：Vector 模式下罗盘与地图底图对比度不足，可读性差
- **解决**：2 倍半径的浅色径向渐变背景（白色系），提高对比度
- **效果**：`rgba(255,255,255, 0.45 → 0.25 → 0.10 → 0)` 从中心向外渐变

### 4. HUD 模式位置调整
- **问题**：HUD 模式固定在右下角，位置不合适
- **解决**：调整为地图几何中心显示
- **实现**：使用 `left: 50%; top: 50%; transform: translate(-50%, -50%)` 居中定位

## 🔍 修改原因

1. **半径判断**：用户反馈罗盘在缩放时容易消失，需要更宽松的可见性判断
2. **缩放分级**：固定 `minResolution` 无法适应不同缩放级别的显示需求
3. **显示效果**：Vector 模式下罗盘缺乏视觉层次，需要增强显示效果
4. **HUD 位置**：右下角位置在实际使用中不方便观察，中心位置更合理

- **MapContainer.vue**：地图容器组件（HUD 模式样式）

## 🛠️ 优化解决方案

### 1. 半径判断算法

```typescript
// 计算视图四角到罗盘圆心的距离，取最大值
const corners: [number, number][] = [
    [0, 0],           // 左上
    [width, 0],       // 右上
    [width, height],  // 右下
    [0, height],      // 左下
];

let maxCornerDistanceSq = 0;
for (const [cornerX, cornerY] of corners) {
    const dx = centerX - cornerX;
    const dy = centerY - cornerY;
    const distSq = dx * dx + dy * dy;
    if (distSq > maxCornerDistanceSq) {
        maxCornerDistanceSq = distSq;
    }
}

// 罗盘半径 >= 视图四至到圆心最大距离的 1.5 倍时可见
const threshold = Math.sqrt(maxCornerDistanceSq) * 1.5;
return radiusPx >= threshold;
```

### 2. 缩放等级动态判断

```typescript
// 基于当前缩放等级 + 3 级计算最大可见分辨率
const view = this.map.getView?.();
const currentResolution = Number(view?.getResolution?.());
if (Number.isFinite(currentResolution) && currentResolution > 0) {
    const maxVisibleResolution = currentResolution * 8;
    this.layer.setMaxResolution(maxVisibleResolution);
}
```

### 3. 渐变背景实现

```typescript
// 渐变背景：模仿 HUD 模式的 radial-gradient 效果
const bgGradient = context.createRadialGradient(0, 0, 0, 0, 0, radiusPx * 1.05);
bgGradient.addColorStop(0, 'rgba(18, 24, 18, 0.42)');
bgGradient.addColorStop(0.58, 'rgba(18, 24, 18, 0.14)');
bgGradient.addColorStop(0.74, 'rgba(18, 24, 18, 0.05)');
bgGradient.addColorStop(1, 'rgba(18, 24, 18, 0)');
context.fillStyle = bgGradient;
context.beginPath();
context.arc(0, 0, radiusPx * 1.05, 0, Math.PI * 2);
context.fill();
```

## 📈 性能指标

- **半径判断**：新算法增加了 4 次距离计算，但避免了不必要的渲染，整体性能持平
- **缩放等级**：动态计算替代固定值，无额外性能开销
- **渐变背景**：Canvas 渐变绘制开销极小，不影响渲染性能
