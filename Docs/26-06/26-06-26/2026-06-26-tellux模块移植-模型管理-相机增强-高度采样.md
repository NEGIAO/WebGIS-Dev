# 2026-06-26 15:30 — tellux 功能模块移植：模型管理、相机增强、高度采样

## 修改内容

从 tellux 项目（Three.js 3D 地理空间引擎）移植三个核心功能模块到 WebGIS_Dev 项目的 Cesium 架构中：

1. **模型管理器** `useCesiumModelManager.js` — 新增
2. **相机控制增强** `useCesiumCameraEnhanced.js` — 新增
3. **高度采样增强** `useCesiumHeightSampler.js` — 新增

## 修改原因

tellux 项目采用了 Manager + Layer 的分层架构，代码职责清晰、生命周期管理完善。WebGIS_Dev 现有的 Cesium 实现中，模型加载散落在 `useCesiumDataImport.js` 中，相机飞行功能简单，高度采样缺乏封装。本次移植将 tellux 的优秀设计模式适配到 Cesium API 上，增强项目的功能完备性和代码可维护性。

## 影响范围

- Cesium 3D 组件层：`frontend/src/components/Cesium/composables/`
- 新增 composable，不影响现有功能，可渐进式集成

## 移植策略

由于 tellux 基于 Three.js 而 WebGIS_Dev 基于 CesiumJS，**无法直接复制代码**。采用以下策略：

| 维度 | 说明 |
|---|---|
| 设计模式移植 | Manager → Map<id, Layer> 架构、生命周期管理 |
| 算法移植 | 缓动函数、弹簧物理（SpringController） |
| API 适配 | THREE.PerspectiveCamera → Cesium.Camera、GLTFLoader → Cesium.Model |

## 模块详情

### 1. useCesiumModelManager.js（306行）

参考 tellux `ModelManager + GltfModelLayer` 架构：

- **addModel(options)** — 加载 glTF/GLB 到地理坐标位置，支持 heading/pitch/roll 姿态和缩放
- **addModelFromFile(file)** — 从本地文件加载，管理 Object URL 生命周期
- **removeModel(id)** — 释放场景资源 + 内部资源
- **updateModel(id, updates)** — 动态更新位置/姿态/缩放
- **flyToModel(id)** — 相机飞行到模型附近
- **playAnimation / stopAnimation** — 动画播放控制
- 响应式状态：`models`、`loadingCount`、`readyCount`

### 2. useCesiumCameraEnhanced.js（430行）

参考 tellux `Camera + SpringControl + TargetFlightController`：

- **EasingFunctions** — 10 种缓动曲线（线性、二次/三次/正弦 缓入缓出）
- **SpringController 类** — 阻尼谐振子弹簧物理控制器
- **enhancedFlyTo(options)** — 增强飞行：自定义缓动、角度插值、飞行取消、每帧回调
- **flyToPosition / flyToAndLookAt / flyToRectangle** — 快捷飞行方法
- **springToHeight** — 弹簧物理高度过渡
- **getCameraState / getFormattedPosition** — 相机状态查询
- 角度插值处理 0° ↔ 360° 环绕

### 3. useCesiumHeightSampler.js（360行）

参考 tellux `HeightSampler + CartographicPicker`：

- **pickCartographic(screenPos)** — 屏幕坐标 → 地理坐标（raycast + 降级策略）
- **pickCartographicWithRetry** — 带重试的拾取（地形瓦片未完全加载时使用）
- **sampleHeight(position)** — 单点同步高度采样
- **sampleHeightMostDetailed(positions)** — 批量异步最详细采样（缓存 + 进度回调 + 降级）
- **queryHeightRange(rectangle)** — 区域高度范围查询（min/max/avg）
- 结果缓存：5 分钟 TTL，经纬度精度 5 位小数

## 性能指标

- 所有模块均为轻量级 composable，按需引入，不增加初始加载体积
- 高度采样缓存可避免重复请求地形 tile

## 测试方案

1. 在 CesiumContainer.vue 中引入 composable 并调用 addModel 测试模型加载
2. 使用 enhancedFlyTo 替代原生 flyTo 测试飞行动画
3. 使用 sampleHeightMostDetailed 在不同地形区域测试高度采样精度
4. 检查 ESLint 无报错
