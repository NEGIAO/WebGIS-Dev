# v3.2.6 可复用 ExtentPicker 框选组件变更日志

- **日期和时间**：2026-06-05 00:20
- **修改内容**：
  1. 新增 `ExtentPicker.vue` 通用框选范围组件（开始框选/重新框选/清除选区/状态提示/坐标标签）
  2. `SpatialAnalysisPanel.vue` 渔网分析的"地图选取"按钮替换为 `ExtentPicker`（新增清除功能 + 状态提示）
  3. `MapDownloader.vue` 框选 UI 替换为 `ExtentPicker`（统一交互体验）
  4. Code Review 修复：`router.py` 复用 `_get_transformer` 缓存、`utils.py` 循环变量遮蔽、`MAX_GRID_CELLS` 提取为共享常量

- **修改原因**：
  1. 原 `pickBoxExtent` 仅是底层 Promise API，缺少完整的交互 UI（开始/重新/清除/提示）
  2. MapDownloader 和 SpatialAnalysisPanel 各自内联了框选按钮，逻辑重复
  3. 需要一个可复用的框选组件，供所有需要范围选取的功能统一使用

- **影响范围**：前端 Common 组件库、空间分析面板、底图下载面板

- **优化解决方案**：

  ### ExtentPicker 组件设计

  - **Props**：`hasExtent`（Boolean）、`disabled`（Boolean）、`extentLabel`（String，坐标描述）
  - **Events**：`pick-extent`（开始/重新框选）、`clear-extent`（清除选区）
  - **UI 状态**：
    - 未框选：绿色「开始框选」按钮 + 灰色提示"在地图上拖拽矩形框选范围"
    - 已框选：绿色「重新框选」按钮 + 红色「清除选区」按钮 + 绿色"✓ 已框选范围" + 坐标标签
  - **样式**：CSS 变量驱动，跟随主题系统，清除按钮独立红色样式

  ### 复用集成

  | 组件 | 原实现 | 改为 |
  |------|--------|------|
  | SpatialAnalysisPanel | 内联"地图选取"按钮 | `ExtentPicker` + `clearFishnetExtent()` |
  | MapDownloader | 内联"地图框选范围"/"重新框选"/"清除选区" 3 个按钮 + hint | `ExtentPicker` |

  ### Code Review 修复

  - `router.py`：bbox 转换从 `Transformer.from_crs()` 改为 `_get_transformer()` 缓存复用
  - `utils.py:_reproject_features`：`feat = {**feat, ...}` 变量遮蔽 → `new_feat = {**feat, ...}; out.append(new_feat); continue`
  - `MAX_GRID_CELLS`：从 `aggregation.py` 和 `fishnet.py` 各自定义 → 提取到 `utils.py` 共享常量

- **测试方案**：
  1. 渔网分析：点击「开始框选」→ 拖拽 → 坐标自动填充 + 提示"✓ 已框选范围" + 显示坐标
  2. 渔网分析：点击「清除选区」→ 四至清空 + 提示恢复"在地图上拖拽矩形框选范围"
  3. 渔网分析：已有范围时点击「重新框选」→ 重新拖拽 → 范围更新
  4. 底图下载：框选交互与渔网分析一致，体验统一
