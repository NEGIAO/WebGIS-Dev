# v3.2.3 渔网分析功能变更日志

- **日期和时间**：2026-06-04 22:30
- **修改内容**：
  1. 后端新增 `fishnet` 渔网分析操作（`do_fishnet()`），支持在指定四至范围内按固定间距（米）生成规则网格
  2. 前端空间分析面板新增"渔网分析"工具，支持手动输入/地图框选/当前视图三种范围获取方式
  3. 抽取通用 `pickBoxExtent()` 框选函数，从 MapDownloader 的下载专用逻辑中解耦，供多个组件复用
  4. 支持输出面（Polygon）/ 线（LineString）两种几何类型，可选创建每个网格中心点
  5. 渔网结果自动加入 TOC 图层管理，命名规则：`渔网_{网格大小}m_{面/线/点}`

- **修改原因**：
  1. 空间分析工具集中缺少渔网/格网分析能力，用户需要在指定范围内生成规则网格用于空间统计和分析
  2. MapDownloader 的框选范围逻辑与下载功能耦合，无法被其他组件（如渔网分析）复用
  3. 渔网分析的网格大小以米为单位，需要在 EPSG:3857 平面坐标系下精确计算

- **影响范围**：后端空间分析 API（`api/spatial.py`）、前端空间分析面板（`SpatialAnalysisPanel.vue`）、地图交互选点（`useMapInteractionPickers.js`）、空间分析调度（`useSpatialAnalysis.js`）、MapContainer、ControlsPanel、HomeView

- **优化解决方案**：

  ### 1. 后端渔网分析（`backend/api/spatial.py`）

  - `SpatialAnalysisRequest` 模型新增字段：`grid_size_meters`（网格大小，米）、`geometry_type`（polygon/line）、`create_center_points`（是否创建中心点）
  - 新增 `do_fishnet()` 函数：
    - 使用 `pyproj.Transformer` 将 bbox 从 EPSG:4326 转到 EPSG:3857
    - 在 3857 平面坐标系下按 `grid_size_meters` 精确计算网格（整数行列循环，避免浮点累积误差）
    - 将每个网格四角从 3857 转回 4326，根据 `geometry_type` 生成 Polygon 或 LineString
    - 可选在每个网格中心创建 Point 要素
    - `MAX_GRID_CELLS = 100000` 安全限制
  - 主端点新增 `fishnet` 分支，`features_a` 为空集合时跳过几何校验

  ### 2. 通用框选取能力（`useMapInteractionPickers.js`）

  - 新增 `pickBoxExtent(options?)` 函数：复用 DragBox 交互逻辑，返回 `Promise<{ extent: [4326], crs }>`
  - 与 `pickDownloadExtent` 共享预览图层（`ensurePreviewLayer`/`updatePreviewGeometry`），但使用独立的 pending ref 和 interaction 变量
  - `options.showOverlay` 控制完成后是否显示持久化覆盖层
  - `cancelBoxExtentPick()` 取消函数
  - `disposeAll()` 清理新增状态

  ### 3. 前端渔网分析面板（`SpatialAnalysisPanel.vue`）

  - 新增 `fishnet` 工具：四至输入框（4 个数字输入）、"当前视图"按钮、"地图选取"按钮、网格大小（米）、几何类型（面/线 radio）、创建中心点（checkbox）
  - `pickExtent` prop 接收异步框选函数
  - `handlePickExtent()` 调用 `props.pickExtent()` 并填充 bbox
  - `fillFishnetBboxFromMapExtent()` 从当前视图获取范围
  - `runFishnet()` emit 分析事件

  ### 4. 事件通道（ControlsPanel → HomeView → MapContainer）

  - `ControlsPanel.vue`：新增 `pickExtent` prop，透传给 `SpatialAnalysisPanel`
  - `HomeView.vue`：新增 `pickExtentForAnalysis()` 异步函数，调用 `mapContainerRef.value?.pickBoxExtent?.()`，通过 `:pick-extent` 传给 ControlsPanel
  - `MapContainer.vue`：`defineExpose` 新增 `pickBoxExtent`

  ### 5. 分析调度（`useSpatialAnalysis.js`）

  - 新增 `type === 'fishnet'` 分支：不依赖输入图层，直接构造 payload（`operation: 'fishnet'`, `bbox`, `grid_size_meters`, `geometry_type`, `create_center_points`）
  - 结果处理：主图层（面/线）`createManagedVectorLayer` + 如果有 `center_points` 则额外创建点图层
  - fishnet 分支提前 return，不走通用的结果处理流程

- **测试方案**：
  1. 手动输入四至范围 + 设置网格大小 500m + 选择"面" → 执行 → 地图显示规则网格面图层，TOC 命名为 `渔网_500m_面`
  2. 点击"地图选取" → 拖拽框选 → 四至自动填充 → 执行 → 结果正确
  3. 点击"当前视图" → 四至自动填充 → 执行 → 结果正确
  4. 选择"线"类型 → 执行 → 结果为 LineString 几何
  5. 勾选"创建渔网中心点" → 执行 → 额外生成 `渔网_500m_点` 图层
  6. 设置极大范围 + 极小网格 → 触发 MAX_GRID_CELLS 限制 → 显示错误提示
  7. 下载框选功能不受影响（`pickDownloadExtent` 行为不变）
