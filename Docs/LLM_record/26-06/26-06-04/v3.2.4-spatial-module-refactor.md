# v3.2.4 空间分析模块拆分变更日志

- **日期和时间**：2026-06-04 23:00
- **修改内容**：
  1. 将单文件 `api/spatial.py`（784 行）拆分为模块化目录结构 `api/spatial/`
  2. 公共工具函数提取到 `utils.py`（坐标转换、几何格式转换、顶点统计）
  3. Pydantic 模型提取到 `models.py`（GeoJSONFeatureCollection、SpatialAnalysisRequest）
  4. 8 个分析操作各自独立为 `operations/` 子模块文件
  5. 路由 + dispatch 逻辑提取到 `router.py`
  6. `__init__.py` 门面保持 `from api.spatial import router` 完全兼容

- **修改原因**：
  1. 单文件 784 行不便维护，新增操作（如渔网分析）需要在大文件中定位插入点
  2. 所有操作耦合在同一文件，修改某个操作可能影响其他操作的阅读和理解
  3. 参考 `api/auth/` 模块的组织方式，统一项目结构规范

- **影响范围**：后端空间分析 API（`backend/api/spatial/`），前端无需改动

- **优化解决方案**：

  ### 目录结构

  ```
  backend/api/spatial/
  ├── __init__.py              # 门面：re-export router
  ├── models.py                # Pydantic 请求/响应模型
  ├── utils.py                 # 公共工具函数
  ├── router.py                # FastAPI 路由 + 端点分发
  └── operations/              # 分析操作实现
      ├── __init__.py           # re-export 所有 do_* 函数
      ├── buffer.py             # do_buffer
      ├── overlay.py            # do_intersection, do_union, do_difference
      ├── convex_hull.py        # do_convex_hull
      ├── voronoi.py            # do_voronoi
      ├── aggregation.py        # do_spatial_aggregation
      ├── multi_ring_buffer.py  # do_multi_ring_buffer
      ├── simplify.py           # do_simplify
      └── fishnet.py            # do_fishnet
  ```

  ### 拆分原则

  - 每个 operation 文件通过 `from ..utils import ...` 引用公共函数
  - `operations/__init__.py` re-export 所有 `do_*` 函数，`router.py` 通过 `from .operations import ...` 统一导入
  - `__init__.py` 门面只 re-export `router`，保持外部 `from api.spatial import router` 路径不变
  - overlay 的三个操作（intersection/union/difference）共享 `geoms_b` 参数模式，归为一组

- **测试方案**：
  1. 启动后端服务 → 无 import 错误
  2. 调用 `/api/v1/spatial/analysis` 各操作 → 返回结果与拆分前一致
  3. 前端空间分析功能不受影响
