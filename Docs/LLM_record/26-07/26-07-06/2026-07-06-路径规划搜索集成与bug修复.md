# 路径规划搜索集成 + 注记图层 HD 兼容 + 错误处理优化

**日期时间**：2026-07-06
**主题**：驾车/公交规划面板集成天地图搜索、注记图层高清瓦片兼容修复、错误处理与日志优化
**版本**：WebGIS V3.3.16

---

## 一、问题核心症状

### 1.1 驾车/公交规划仅支持地图选点
驾车规划（`DrivingPlannerPanel.vue`）与公交规划（`BusPlannerPanel.vue`）的起点/终点输入，原先仅支持交互式地图点击选点（通过 `MapPointPickerCard.vue` 的地图拾取按钮）。用户无法通过关键词搜索来设定起终点，使用体验受限。

### 1.2 天地图搜索 API 已有封装但未接入
项目中 `api/locationSearch.js` 已封装 `fetchLocationResultsByService` 统一入口，支持天地图/Nominatim/高德三种服务，`LocationSearch.vue` 已实现完整搜索组件。但路线规划面板未复用该搜索能力。

### 1.3 注记图层在 HD 模式下文字过小
启用高清瓦片渲染（`tileHDRendering=true`）时，`buildRasterBasemapSource` 对所有 raster source 设置 `zDirection` 为 `alwaysPickFinerZoom`（非整数 zoom 取上层瓦片）。注记图层（`category='label'`）的文字在取上层瓦片后因分辨率翻倍而过小，影响可读性。

### 1.4 错误处理链路不完善
- 天地图 Token 缺失时抛出非语义化错误，用户无法明确知道需要配置 Token
- `e instanceof TypeError` 与真实网络错误的判断不准确（渲染链路 TypeError 被误捕）
- 公交/驾车规划的调试信息与地图渲染顺序不合理，渲染失败后调试信息也被丢失

## 二、根本原因分析（事件逻辑链条）

### 2.1 搜索功能未集成路线规划

`MapPointPickerCard.vue` 是 `DrivingPlannerPanel.vue` 和 `BusPlannerPanel.vue` 共享的起点/终点选择组件。原有设计仅提供了「地图拾取」交互（`emit('pick-start')` / `emit('pick-end')`），没有搜索输入框。

前端 `api/locationSearch.js` 已封装完整的 `fetchLocationResultsByService({ service: 'tianditu', keywords, page, pageSize, tiandituTk, signal })` 调用链路，包括天地图 `https://api.tianditu.gov.cn/v2/search` 接口的 keyword/mapBound/queryType 参数组织。该功能在 `LocationSearch.vue` 中已有成功应用，但未集成到 `MapPointPickerCard.vue`。

### 2.2 注记图层 HD 兼容缺失

`buildRasterBasemapSource`（`basemapLayerFactory.js`）是所有 raster 底图 source 创建/替换的唯一入口。HD 开关启用时无条件设置 `zDirection = alwaysPickFinerZoom`。注记图层的文字在 256px 瓦片中被设计为特定字号，当强制取上层（z+1）瓦片时，文字物理尺寸减半，在屏幕上显示过小。

`basemapConfig.ts` 中 4 个注记图层源（天地图 cia/cva、GeoVIS cia、高德注记）的 `createSource` 直接返回 `prioritizeTileSourceRequest(new XYZ(...))`，没有 `skipHighResTile` 标记。`buildRasterBasemapSource` 也没有检查 `skipHighResTile` 标志。

### 2.3 错误处理设计缺陷

驾车规划 `startDriveSearch` 中：
- 使用固定 `YOUR_TIANDITU_TK` 占位符作为 Token 默认值，无 Token 时误导用户
- `e instanceof TypeError` 判断网络错误会误捕渲染链路的 TypeError（如 `drawDriveRouteOnMap` 中 `forEachFeatureAtPixel` 返回值语义问题导致的错误）
- 调试信息更新（`debug.rawDistance` 等）放在地图渲染之后，地图渲染失败时调试信息也被丢失

公交规划 `startTransitPlan` 中：
- Token 校验缺失，空 Token 直接发送请求到天地图 API
- `err instanceof TypeError` 同样存在误捕问题

## 四、优化解决方案

### 4.1 路径规划面板集成天地图搜索

**方案**：将 `MapPointPickerCard.vue` 改造为同时支持「地图选点」与「关键词搜索」的混合输入组件。

具体实现：
1. **新增 props**：`tiandituTk`（天地图 Token）、`searchPlaceholder`（搜索框占位符文本）
2. **新增 emits**：`select-start-result` / `select-end-result`（搜索选中的结果，含 lng/lat/address）
3. **搜索逻辑**：使用 `fetchLocationResultsByService({ service: 'tianditu', ... })`，400ms 防抖输入
4. **AbortController 防竞态**：每次新输入时 abort 前一个进行中的请求，防止乱序覆盖
5. **键盘导航**：上下方向键高亮、Enter 确认选择
6. **搜索项截断预览**：`display_name` CSS 单行截断，`:title` 属性悬浮显示全文
7. **模板逻辑修复**：`v-else` → `v-else-if="!hasStart"` / `v-else-if="!hasEnd"`，避免坐标显示后错误显示空白态

在 `DrivingPlannerPanel.vue` 中：
- 传入 `:tianditu-tk="token"`，处理 `@select-start-result` / `@select-end-result`
- `onSelectStartResult` / `onSelectEndResult` 设置起终点坐标和地址

在 `BusPlannerPanel.vue` 中：
- 同样传入 `:tianditu-tk="token"`，处理搜索事件

### 4.2 注记图层 HD 兼容

**方案**：为注记图层 source 添加 `skipHighResTile` 自定义属性标记，`buildRasterBasemapSource` 检查该标记跳过 `zDirection` 设置。

具体实现：
1. 新建 `withSkipHighResTile<T>(src: T)` 辅助函数，通过 `Object.assign(src, { skipHighResTile: true })` 注入自定义属性
2. 在 `LAYER_SOURCE_DEFINITIONS` 中 4 个 `category: 'label'` 图层的 `createSource` 内部使用 `withSkipHighResTile` 包裹
3. 在 `buildRasterBasemapSource` 中增加条件：`if (tileHDRendering.value && !rawSource.skipHighResTile)`

### 4.3 错误处理与链路优化

**驾车规划**：
1. 新增 `TokenMissingError` 自定义错误类，Token 空时抛语义化错误，提示用户配置 Token
2. 移除 `e instanceof TypeError` 判断，改用 `/failed\s+to\s+fetch/i.test(rawMessage)` 精准识别网络错误
3. 先更新调试信息（`debug.rawDistance` 等），再执行地图渲染（可能抛出错误但调试信息已保留）
4. 非网络错误时 `console.error` 记录详细信息

**公交规划**：
1. Token 前置校验：在构建请求 URL 前检查 `tk` 是否为空，空则抛语义化错误
2. 移除 `err instanceof TypeError`，改用 `/failed\s+to\s+fetch/i.test(String(rawMessage))`
3. 非网络错误时 `console.error` 记录详细信息

## 五、实施步骤

### 5.1 搜索集成

1. **`MapPointPickerCard.vue`**（核心修改）：
   - 新增 `tiandituTk`、`searchPlaceholder` props
   - 新增 `select-start-result`、`select-end-result` emits
   - 新增 `startKeyword`/`endKeyword`、`startResults`/`endResults`、`showStartDropdown`/`showEndDropdown`、`startSearching`/`endSearching`、`startHighlightIdx`/`endHighlightIdx` 等响应式状态
   - 新增 `doStartSearch()` / `doEndSearch()` 异步搜索函数，调用 `fetchLocationResultsByService`
   - 新增 `onStartSearchInput()` / `onEndSearchInput()` 防抖 + AbortController 竞态保护
   - 新增键盘/鼠标导航支持
   - 模板层：起点/终点坐标卡片上方插入搜索输入框区域 + 下拉结果列表
   - 修复 `v-else` → `v-else-if="!hasStart"` / `v-else-if="!hasEnd"`

2. **`DrivingPlannerPanel.vue`**：
   - 传入 `:tianditu-tk="token"` 到 MapPointPickerCard
   - 新增 `onSelectStartResult` / `onSelectEndResult` 处理函数

3. **`BusPlannerPanel.vue`**：
   - 传入 `:tianditu-tk="token"` 到 MapPointPickerCard
   - 新增 `onSelectStartResult` / `onSelectEndResult` 处理函数

### 5.2 注记 HD 兼容

1. **`basemapConfig.ts`**：
   - 新建 `withSkipHighResTile<T>` 泛型辅助函数
   - 天地图 cia(`cia_w`)、天地图 cva(`cva_w`)、GeoVIS cia、高德注记 四个 `createSource` 用 `withSkipHighResTile` 包裹

2. **`basemapLayerFactory.js`**：
   - `buildRasterBasemapSource` 增加 `!rawSource.skipHighResTile` 条件判断

### 5.3 错误处理优化

1. **`DrivingPlannerPanel.vue`**：
   - 新增 `TokenMissingError` 类
   - Token 校验改为 `if (!token) throw new TokenMissingError()`
   - 移除 `e instanceof TypeError`，改为 `/failed\s+to\s+fetch/i` + `console.error`
   - 调试信息更新提前到地图渲染之前

2. **`BusPlannerPanel.vue`**：
   - Token 前置校验：构建 URL 前检查 `tk`
   - 移除 `err instanceof TypeError`，改为 `/failed\s+to\s+fetch/i`

### 5.4 Edit 工具重复内容修复

在连续使用 Edit 工具对同一文件（MapPointPickerCard.vue）进行多次 `old_string`/`new_string` 替换时，由于旧字符串匹配精度不足，导致：
- 重复的 `import` 声明（import { computed, ref } from 'vue' 出现 2 次、import { fetchLocationResultsByService } 出现 2 次）
- 重复的 props 声明（`tiandituTk`、`searchPlaceholder` 声明 2 次）
- 重复的 `defineEmits` 声明
- 搜索逻辑代码块四倍重复（start→end→start→end）

修复方式：逐一删除每个重复块，最后 `npm run build` 验证通过。

## 七、测试方案

### 7.1 测试环境
- 前端开发服务器：`npm run dev`
- 浏览器：Chrome 最新稳定版

### 7.2 测试步骤

**搜索集成验证**：
1. 打开驾车规划面板 → 起点输入框输入关键词 → 验证下拉显示搜索结果
2. 选择搜索结果 → 验证坐标和地址正确设置 → 验证地图选点按钮仍可正常使用
3. 终点重复上述步骤
4. 快速输入多个关键词 → 验证仅有最后一次请求生效（无乱序覆盖）
5. 键盘上下方向键切换高亮 → Enter 确认 → 验证选中正确
6. 公交规划面板重复上述验证
7. 输入不存在的关键词 → 验证显示"暂无结果"占位项

**注记 HD 兼容验证**：
1. 启用 HD 开关（`tileHDRendering=true`）
2. 非整数 zoom（如 11.5）→ 验证注记图层文字大小正常，路径图层瓦片更清晰
3. 禁用 HD 开关 → 验证注记图层行为回退到默认

**错误处理验证**：
1. 清空天地图 Token → 驾车规划 → 验证显示 Token 配置提示（非"网络错误"）
2. 恢复 Token → 正常搜索 → 验证功能恢复
3. 公交规划重复 Token 校验验证

- 驾车/公交规划起终点均支持关键词搜索选点
