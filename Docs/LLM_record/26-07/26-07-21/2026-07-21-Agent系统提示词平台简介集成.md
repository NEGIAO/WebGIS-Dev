# 2026-07-21 Agent 系统提示词平台简介集成

## 基本信息
- **日期时间**: 2026-07-21 09:19
- **修改类型**: 功能增强
- **严重程度**: 低（纯提示词增强，不影响现有地图操作链路）

## 事件逻辑链分析

### 核心症状
用户向 AI 助手询问"本平台有什么功能/特色"时，助手只能基于通用知识泛泛而谈，无法准确介绍 WebGIS 3.0 的实际能力（2D/3D 双引擎、数据导入格式、空间分析、三维特效等），因为系统提示词中只有工具调用说明，没有任何平台背景上下文。

### 根因
`frontend/src/constants/agentToolsSchema.js` 的 `buildSystemPromptWithTools()` 返回的系统提示词仅包含三个工具（zoom_to_extent / search_and_zoom / switch_basemap）的调用规范，缺少平台身份与功能概览，导致 LLM 无据可依。

### 受影响模块
- `frontend/src/constants/agentToolsSchema.js` — `buildSystemPromptWithTools()` 函数
- `frontend/src/components/Chat/ChatPanelContent.vue` — `sendMessage()` 第 1399 行调用该函数，经 `_callLLMAPI()` 注入三种 AI 模式（默认 AI / 个人 Key / 后端代理）

### 解决方案
在 `buildSystemPromptWithTools()` 返回的提示词中，于"可用工具"章节之前插入「平台简介」段落（约 350 字），覆盖 8 个维度：2D/3D 双引擎、底图源、数据导入格式、空间分析、路径规划、三维特效、实用工具、账号体系。并附加行为指引：用户询问平台功能时简洁回答，涉及具体操作时引导使用对应面板。

## 修改内容
1. **`buildSystemPromptWithTools()` 开头身份句扩写**：由"你是一个 WebGIS 地图助手"改为"你是一个 WebGIS 地图助手，运行在「WebGIS 3.0」平台上"
2. **新增「平台简介」章节**：以 Markdown 列表形式注入 8 条平台特色，供 LLM 回答平台相关问题时引用
3. **新增行为指引句**：明确"回答平台相关问题时简洁明了；涉及具体操作时引导用户使用对应面板或工具"

## 修改文件路径
1. `frontend/src/constants/agentToolsSchema.js` — `buildSystemPromptWithTools()`（第 142-159 行）

## 影响范围
- **三种 AI 模式全覆盖**：`buildSystemPromptWithTools()` 在 `sendMessage()` 中被调用，经 `_injectToolPromptIntoHistory()` 作为首条伪 user 消息注入 history，默认 AI 模式、个人 Key 模式、后端代理模式均生效
- **平台问答能力**：用户询问"这个平台有什么功能""支持哪些格式""有哪些特效"等问题时，助手可基于平台简介准确作答
- **工具调用链路不变**：原有三个工具的调用规范、XYZ URL 表、注意事项均未改动，地图操作行为不受影响
- **Token 开销**：平台简介约 350 字（≈500-700 token），随每次请求的 history 一并发送（无状态 API 架构所致），为固定增量成本

## 测试方案
1. 打开 ChatPanel，发送"这个平台有什么功能？"，验证助手能列出 2D/3D 双引擎、数据导入、空间分析、三维特效等特色
2. 发送"支持导入哪些数据格式？"，验证助手回答 GeoJSON/KML/KMZ/SHP/GLB/CZML/3D Tiles
3. 发送"切换到高德卫星"，验证工具调用链路不受平台简介影响，底图正常切换
4. 发送"定位到北京大学"，验证 search_and_zoom 工具正常触发
5. 分别切换默认 AI / 个人 Key / 后端代理三种模式，验证平台简介均生效

## 性能指标
单次请求新增约 500-700 token 系统上下文（平台简介为固定开销，不随对话轮次增长）。相对现有工具提示词（含 20+ 条 XYZ URL 表，约 1500-2500 token）为中等增量；随对话历史增长，该固定开销占比被稀释。无前端运行时性能影响（仅字符串拼接，O(1)）。
