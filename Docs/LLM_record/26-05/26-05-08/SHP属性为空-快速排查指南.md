# SHP 属性为空 - 快速故障排查指南

> 如果上传 SHP 后属性仍然为空，请按照本指南逐步排查

---

## 🔧 快速排查流程

### 步骤 1: 检查上传的文件

```
✓ 必须有的文件:
  - .shp (几何数据) ← 最关键
  - .dbf (属性表)   ← 需要有

✓ 可选但推荐:
  - .shx (索引)
  - .prj (坐标系)
  - .cpg (编码)
```

**检查方法**:
1. 打开文件浏览器 `c:\Users\NEGIAO\Desktop\aaa\`
2. 查看是否同时有 `.shp` 和 `.dbf` 文件
3. 验证它们的文件名前缀相同（例如 `测试shp.shp` 和 `测试shp.dbf`）

---

### 步骤 2: 查看浏览器控制台日志

打开浏览器开发者工具 (`F12` 或 `Ctrl+Shift+I`)，查看 **Console** 标签。

#### ✓ 正常情况的日志
```
[SHP] 开始属性解析链路 - {
  hasDbf: true,
  hasCpg: true,
  ...
}
[dbfParser] 检测编码: GBK (CPG: GBK, LDID: 0x4D)
[dbfParser] 解析 15 个字段，预期 5000 条记录
[dbfParser] 成功解析 4977 条有效属性记录
[SHP] ✓ 完成属性增强
  - 记录数: 4977
  - 字段数: 15
  - 编码: GBK
```

#### ✗ 属性为空的常见日志
```
[SHP] 检测到单个 .shp 文件上传，属性信息可能不完整。
[SHP] 未提供 DBF 文件，仅导入几何数据
```

---

### 步骤 3: 根据日志内容排查

#### 情况 A: 没有 DBF 文件
```
日志: "未提供 DBF 文件，仅导入几何数据"
```

**原因**: 上传时没有包含 `.dbf` 文件

**解决方案**:
- [ ] 确认存在 `.dbf` 文件
- [ ] 与 `.shp` 文件一起上传
- [ ] 或打包成 ZIP 后上传

#### 情况 B: DBF 文件存在但解析失败
```
日志: "自定义 DBF 解析失败，继续使用 shpjs 提供的属性"
错误信息: "DBF 缓冲区过小" 或 "编码检测失败"
```

**可能原因**:
- [ ] DBF 文件格式不标准
- [ ] 文件损坏或截断
- [ ] 文件编码不被识别

**排查方法**:
1. 用文本编辑器打开 `.dbf` 文件（或用 16 进制编辑器）
2. 检查文件大小是否合理（不能只有几个字节）
3. 尝试上传时同时包含 `.cpg` 文件（如果存在）

#### 情况 C: 编码检测失败
```
日志: "检测编码: utf-8 (CPG: none, LDID: 0x00)"
但属性仍然是乱码或空
```

**原因**: 自动编码检测失败，默认使用了 UTF-8

**排查方法**:
1. 检查是否存在 `.cpg` 文件
2. 如果没有，用十六进制编辑器查看 `.dbf` 文件第 30 个字节
   - `0x4D` 或 `0x7A` = GBK (中文常见)
   - `0x4F` = Big5
   - `0x03` = CP1252
3. 如果不确定，尝试创建 `.cpg` 文件：
   ```
   文件名: 测试shp.cpg
   内容: GBK
   （只需5个字节的 ASCII 文本）
   ```

#### 情况 D: DBF 有记录但没有字段定义
```
日志: "未找到任何有效的 DBF 字段定义"
```

**原因**: DBF 文件头格式非标准，字段定义位置或结构不符合规范

**排查方法**:
1. 用 ArcGIS、QGIS 等 GIS 软件打开 SHP，检查属性是否正常
2. 如果在 GIS 软件中也无法打开，说明 SHP 本身有问题
3. 尝试重新导出 SHP 文件

## 📋 完整检查清单

```
□ 步骤 1: 确认文件
  □ 存在 .shp 文件
  □ 存在 .dbf 文件
  □ 文件前缀相同（例如都是 "测试shp.*"）
  □ 文件大小正常（.dbf 通常 > 100KB）

□ 步骤 2: 查看浏览器日志
  □ 打开 F12 控制台
  □ 查找 [SHP] 或 [dbfParser] 日志
  □ 检查是否有错误信息

□ 步骤 3: 根据日志排查
  □ 如果显示 "未提供 DBF"：重新上传同名 DBF
  □ 如果显示 "解析失败"：检查文件是否损坏
  □ 如果显示 "编码检测失败"：创建 .cpg 文件或指定编码

□ 步骤 4: 验证结果
  □ 上传后在属性表中查看是否有数据
  □ 检查中文属性是否显示正常（不乱码）
  □ 验证数值和日期字段是否正确解析
```

## 🆘 仍未解决？

如果按照上述步骤仍未解决，请提供以下信息:

1. **收集日志**:
   - 打开浏览器控制台
   - 完整复制 `[SHP]` 和 `[dbfParser]` 相关的所有日志
   - 粘贴到文本文件

2. **收集文件信息**:
   ```bash
   # 在 PowerShell 中运行
   Get-ChildItem "c:\Users\NEGIAO\Desktop\aaa\" | 
     Select-Object Name, Length | 
     Format-List
   ```
   - 记录所有 SHP 相关文件的大小

3. **文件样本**:
   - 如果可能，上传一个小的测试 SHP 文件（几个 MB）
   - 标注文件的原始编码（如 GBK、UTF-8 等）

4. **环境信息**:
   - 浏览器版本（Chrome/Firefox/Edge）
   - Windows 系统语言（中文、英文等）

## 💡 高级诊断技巧

### 使用测试脚本验证
```javascript
// 在浏览器控制台中粘贴并运行
ShpTestSuite.validateImportedData(features)
```

（如果脚本已加载）

### 用 Python 验证 DBF 文件

```python
# 需要 Python 3.6+ 和 shapefile 库
# 安装: pip install shapefile

import shapefile

# 打开 SHP 文件
sf = shapefile.Reader(r"c:\Users\NEGIAO\Desktop\aaa\测试shp.shp")

# 检查几何
print(f"Shape 数量: {len(sf.shapes())}")

# 检查属性
print(f"字段: {[field[0] for field in sf.fields[1:]]}")

# 读取第一条记录
record = sf.record(0)
print(f"第一条属性: {record}")

# 检查编码
dbf = sf.dbf
print(f"DBF 版本: {dbf.version}")
print(f"LDID: 0x{dbf.dbf_header[0x1D]:02X}")
```
