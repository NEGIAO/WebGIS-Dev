## 📋 功能测试清单

### 测试 1：Native 模式基本流程

**目标**: 验证浏览器托管模式的完整下载流程  
**前置条件**: 后端和前端都已正常启动  
**步骤**:

```
1. 打开 MapDownloader 组件
   └─ 验证：下载模式默认为 "浏览器托管"
   
2. 填写参数
   ├─ 底图源：选择任意支持的底图
   ├─ 分辨率：10 m
   ├─ BBox：使用默认值或手动输入 [116.2, 39.8, 116.3, 39.9]
   └─ 验证：所有字段正常显示

3. 点击"开始下载"
   └─ 验证：任务提交成功，出现任务 ID

4. 等待后端生成
   ├─ 后端进度条更新：0% → 100%
   ├─ 状态变化：pending → downloading → stitching → success
   └─ 验证：无本地传输进度卡片显示

5. 自动下载触发
   ├─ 浏览器下载栏弹出
   ├─ 文件名：basemap_<task_id>.tif
   └─ 验证：文件大小合理（与下载范围相符）

6. 下载完成
   ├─ 浏览器显示"已完成"或文件保存到默认目录
   ├─ 页面提示："下载已启动，文件将在浏览器中下载"
   └─ 验证：即使关闭下载弹窗，浏览器下载继续进行
```

**预期结果**: ✅ 通过
- 文件完整下载，无内存溢出
- 倒计时不显示（Native 模式特性）

---

### 测试 2：Progressive 模式基本流程

**目标**: 验证前端可视化模式的进度显示  
**前置条件**: 完成测试 1  
**步骤**:

```
1. 切换下载模式
   └─ 选择"前端可视化（测试）"

2. 重复测试 1 的步骤 2-3

3. 等待后端生成（同上）
   └─ 验证：仍有"后端生成进度"卡片

4. 下载完成时，自动进入 downloadFileToLocal
   ├─ 显示"本地传输进度"卡片
   ├─ 实时显示：已传输 / 总大小（byte）
   ├─ 倒计时显示：X分 Y秒
   └─ 验证：进度条平滑更新

5. 下载完成
   ├─ 进度条达到 100%
   ├─ 显示提示："文件成功下载到本地！"
   └─ 验证：文件保存成功
```

**预期结果**: ✅ 通过
- 进度条实时更新
- 倒计时正常工作
- 文件完整保存

### 测试 3：Token 过期验证

**目标**: 验证 token 机制的安全性  
**前置条件**: 了解任务 ID 和 token 值  
**步骤**:

```
1. 完成测试 1，获得成功的任务

2. 在浏览器控制台运行
   ├─ 获取当前 task_id: const taskId = '...'
   ├─ 获取 store 中的 downloadToken
   └─ 手动构建错误的 token URL

3. 直接访问错误的下载 URL
   └─ URL 示例：http://localhost:8000/api/download/tasks/<id>/file?token=invalid_token

4. 验证响应
   └─ 预期状态码：401 Unauthorized
   └─ 错误信息：包含 "Download token is invalid or expired"
```

**预期结果**: ✅ 通过
- Token 验证正常工作
- 非法 Token 被正确拒绝

---

### 测试 4：大文件下载稳定性

**目标**: 验证大文件场景下的内存使用  
**前置条件**: 选择较大的下载范围  
**步骤**:

```
1. 配置大下载范围
   ├─ BBox：[116, 39, 117, 40]（更大的地理范围）
   ├─ 分辨率：5 m（更高精度 → 更大文件）
   └─ 预期文件大小：> 500MB

2. Native 模式下载
   ├─ 使用系统监控工具（如 Task Manager）观察内存占用
   ├─ 内存占用应保持 < 100MB（不包括浏览器自身）
   └─ 验证：无明显内存峰值

3. Progressive 模式下载
   ├─ 观察内存占用
   ├─ 预期：可能达到 100-500MB（因为要加载到内存）
   └─ 验证：下载完成后内存释放
```

**预期结果**: ✅ 通过
- Native 模式内存占用显著低于 Progressive 模式
- 下载不会导致浏览器崩溃


**预期结果**: ✅ 通过

- 不会混淆两种下载方式

### 测试 6：任务过期处理

**目标**: 验证任务过期后的行为  
**前置条件**: 了解任务 TTL（30 分钟）  
**步骤**:

```
1. 创建一个下载任务
   └─ 记录创建时间

2. 等待任务完成
   └─ 获得 download_token

3. 在过期前下载（Normal 流程）
   └─ 验证：成功下载

4. 模拟过期
   ├─ 方法 A：等待 30 分钟（太慢，跳过）
   ├─ 方法 B：在后端 _download_tokens 中删除该 token
   └─ 或访问不存在的 task_id

5. 尝试下载过期任务
   └─ 预期响应：410 Gone（或 404 Not Found）
```

**预期结果**: ✅ 通过
- 过期任务无法访问
- 错误提示清晰

## 📞 常见问题

### Q1: Native 模式下如何取消下载？
**A**: 使用浏览器的下载管理器直接取消。我们不提供应用级别的取消机制，因为浏览器已经接管。

### Q2: Progressive 模式的倒计时是用来做什么的？
**A**: 用来提醒用户任务的有效期（30 分钟）。如果倒计时归零，任务将过期无法访问。

### Q3: 如何验证下载的文件完整性？
**A**: 
- 计算 MD5/SHA256 哈希值与原文件对比
- 或在下载的 GeoTIFF 中检查坐标参考系统 (CRS) 和栅格大小

### Q4: 能否并发下载多个任务？
**A**: 可以。每个任务有独立的 token 和生命周期。建议不超过 5 个并发任务（避免网络和服务器压力）。
