# v3.0.11 变更日志

- **日期和时间**：2026-05-30 18:30
- **修改内容**：
  1. 图层操作性能优化（问题二）— 消除拖拽排序、可见性切换、透明度调整、样式应用的卡顿
  2. 下载面板框选范围 UI（问题三）— 框选后地图上持久化显示绿色虚线矩形，支持清除和重新框选
- **修改原因**：
  1. 图层操作全量刷新链路过长：任何操作都走 `refreshLayersState() → refreshLayerInstances() 全量遍历 → emitBaseLayersChangeBatched() → emitUserLayersChange() 序列化全部 features → syncLayers() → layerTree 全部重算`，即使只改一个图层的透明度
  2. 下载框选使用 OL DragBox 交互，boxend 后立即 removeInteraction，矩形瞬间消失，用户无法直观看到已选范围
- **影响范围**：底图控制面板、TOC 图层面板、图层样式系统、下载面板
- **优化解决方案**：

  ### 问题二：图层操作卡顿（5 个瓶颈 → 4 步优化）

  **瓶颈 1**：`handleLayerOrderUpdate` 无差别调用 `refreshLayersState()`
  - **修复**：`useLayerControlHandlers.js` — opacity 只调 `layer.setOpacity()`，visibility 只调 `layer.setVisible()` + 通知底图面板，reorder 才走全量刷新

  **瓶颈 2**：`refreshLayerInstances` 全量遍历无 diff
  - **修复**：`useMapState.js` — 增加值比较，`getVisible()/getZIndex()/getOpacity()` 与目标值相同时跳过 OL 方法调用

  **瓶颈 3**：`hasAttributeFeatures` 每次访问 features 数组
  - **修复**：`layerHelpers.ts` — 优先检查 `featureCount > 0`（O(1)），避免遍历 features

  **瓶颈 4**：`applyManagedLayerStyle` 每次清空 labelStyleCache
  - **修复**：`useManagedLayerStyle.js` — 缓存不存在时才初始化，样式配置变化时显式清空

  ### 问题三：下载框选无 UI

  - `useMapInteractionPickers.js` — 新增 `showExtentOverlay(geometry)` / `clearExtentOverlay()`，框选完成后创建 VectorLayer 持久化显示绿色虚线矩形
  - `MapDownloader.vue` — 新增"清除选区"按钮和选区状态提示
  - 事件链 `MapDownloader → TOCPanel → SidePanel → HomeView → MapContainer` 完整桥接

- **测试方案**：
  1. 加载 2-3 个底图，拖拽排序 → 无明显卡顿
  2. 拖动透明度滑块 → 流畅无延迟
  3. 点击可见性切换 → 即时响应
  4. 应用样式到图层 → 无卡顿
  5. 下载面板点击"地图框选范围" → 拖拽后地图上显示绿色虚线矩形
  6. bbox 输入框同步更新，点击"清除选区"矩形消失
