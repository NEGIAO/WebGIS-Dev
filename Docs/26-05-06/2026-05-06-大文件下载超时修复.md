# 🔧 大文件下载超时修复 (2026-05-06)

**日期/时间**: 2026-05-06 15:45  
**状态**: ✅ 已修复并验证

---

## 📋 问题描述

### 症状
- **现象**: 后端成功生成 GeoTIFF 文件，到最后下载步骤时，前端报错：
  ```
  网络异常,请检查您的连接
  ```
- **触发条件**: 只在**数据量较大**的时候发生
- **影响范围**: 所有大尺寸地图导出任务（高分辨率或大地理范围）

### 根本原因分析

#### 事件链条
1. **后端**：异步任务成功完成 GeoTIFF 生成（可能需要 10-30 秒）
2. **后端**：文件准备就绪，`status = 'success'`，`progress = 100%`
3. **前端**：轮询检测到进度完成，自动触发 `apiDownloadTaskFile(taskId)` 请求
4. **前端 axios**：发起文件下载请求，但使用全局 **8 秒超时**
5. **文件流传输**：大文件（50MB+）传输需要 15-30 秒
6. **超时触发**：axios 在 8 秒后中止请求（`ECONNABORTED` 错误）
7. **错误拦截**：backend.js 响应拦截器将所有 "请求已发出但无响应" 的错误统一报为 "网络异常"
8. **用户看到**：网络错误提示，而不是真正的超时原因

#### 根本原因
- ❌ **问题 1**: 文件下载使用全局 8 秒超时（为 JSON API 设计）
- ❌ **问题 2**: 超时错误被掩盖为"网络异常"，用户无法区分原因

---

## ✅ 解决方案

### 方案设计

#### 1️⃣ 分离文件下载的超时时间
- **原**：所有请求统一 8000ms
- **新**：
  - JSON API 请求：8 秒（保持快速响应）
  - 文件下载请求：300 秒（5 分钟，足以传输大文件）
  - 可配置：支持 `VITE_DOWNLOAD_REQUEST_TIMEOUT` 环境变量

#### 2️⃣ 改进错误拦截，区分超时 vs 网络断开
- 检测 `error.code === 'ECONNABORTED'`（axios 超时标志）
- 检测错误信息中的 "timeout" 关键词
- 显示 "请求超时，请稍后重试" 而非模糊的 "网络异常"

### 实施步骤

#### 修改文件 1: `frontend/src/api/download.js`

**添加下载专用超时常量**
```javascript
/**
 * Download request timeout: 5 minutes (300000ms)
 * Much longer than global 8s timeout since large GeoTIFF files take time to stream
 */
const DOWNLOAD_REQUEST_TIMEOUT = Number(import.meta.env.VITE_DOWNLOAD_REQUEST_TIMEOUT || 300000);
```

**修改 apiDownloadTaskFile 函数**
```javascript
export async function apiDownloadTaskFile(taskId) {
    const safeId = encodeURIComponent(String(taskId || '').trim());
    return backendAPI.get(`/api/download/tasks/${safeId}/file`, {
        responseType: 'blob',
        timeout: DOWNLOAD_REQUEST_TIMEOUT  // Override global 8s timeout for file download
    });
}
```

#### 修改文件 2: `frontend/src/api/backend.js`

**改进错误拦截器的网络错误处理**
```javascript
} else if (error.request) {
  // 请求已发出但没有收到响应
  // 区分超时错误和网络断开
  if (error.code === 'ECONNABORTED' || /timeout/i.test(String(error?.message || ''))) {
    message = '请求超时，请稍后重试'
  } else {
    message = '网络异常,请检查您的连接'
  }
}
```
