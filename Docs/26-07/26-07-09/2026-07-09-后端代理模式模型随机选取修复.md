# 2026-07-09 后端代理模式模型随机选取修复

## 基本信息
- **日期时间**: 2026-07-09 14:30
- **修改类型**: Bug 修复
- **严重程度**: 高（影响所有后端代理模式用户）

## 事件逻辑链分析

### 核心症状
后端代理模式（非默认 AI 模式、非个人 Key 模式）下，ChatPanel 发送消息时使用的模型为随机选取，而非管理员在后台数据库中配置的模型。

### 根因
`backend/api/agent_chat/utils.py` 中的 `_pick_runtime_model` 函数在模型选取优先级链中，将 `random.choice(pool)`（从 `available_models` 随机选）放在优先级 3，而管理员配置的 `provider_model`（`system_config.agent_model`）在优先级 4。只要有可用模型列表，随机选取就永远先行触发，管理员配置被跳过。

### 受影响模块
- `backend/api/agent_chat/utils.py` — `_pick_runtime_model` 函数
- `backend/api/agent_chat/db.py` — `_resolve_effective_agent_runtime_sync` 调用方
- `backend/api/agent_chat/routes.py` — `GET /api/agent/chat/config` 和 `POST /api/agent/chat/completions`
- 前端 `ChatPanelContent.vue` — 通过 `apiAgentGetChatConfig()` 读取模型

### 解决方案
移除 `_pick_runtime_model` 中的 `random.choice(pool)` 逻辑，删除 `import random`（已无其他用途）。新的选取优先级：

1. 用户显式覆盖（`user_override_model`）→ `model_source="user-config"`
2. 用户偏好（`preference_model`）→ `model_source="user-preference"`
3. **管理员配置（`provider_model`）→ `model_source="provider-config"`**（原优先级 3 被直接跳过）
4. 环境默认值（`DEFAULT_AGENT_MODEL`）→ `model_source="env-default"`
5. 空 → `model_source="missing"`

## 修改文件路径
1. `backend/api/agent_chat/utils.py` — 移除 `import random`，`_pick_runtime_model` 移除 `available_models` 参数及 `random.choice` 块
2. `backend/api/agent_chat/db.py` 第 724-728 行 — `_pick_runtime_model` 调用处移除 `available_models` 实参

## 影响范围
- **运行时模型选取**：后端代理模式下的 `/api/agent/chat/config` 和 `/api/agent/chat/completions` 现在返回管理员在 `system_config` 表 `agent_model` 字段中配置的模型
- **前端模型展示**：`ChatPanelContent.vue` 的 `loadAvailableModels()` 通过 `apiAgentGetChatConfig()` 获取模型，后端修复后自动读取正确值，无需前端改动
- **`available_models` 列表**：仍在前端下拉框展示，仅用于用户手动切换模型时的参考，不参与自动选取
- **向后兼容**：`model_source="provider-random"` 不再出现。旧 `model_source` 值（`user-config`、`user-preference`、`provider-config`、`env-default`）保持不变
- **默认 AI 模式和个人 Key 模式**：不受影响，它们使用独立的模型选取路径

## 测试方案
1. 管理员在后台设置 `agent_model` 为特定模型（如 `gpt-4`）
2. 用户以"后端代理模式"打开 ChatPanel，观察状态栏"当前模型"是否显示管理员配置的模型
3. 发送消息，确认实际使用的模型与管理后台一致
4. 验证用户手动在下拉框切换模型后，是否使用用户选择的模型（而非管理员默认）
5. 验证清除个人配置后，恢复使用管理员配置的模型

## 性能指标
无性能影响。移除了一次 `random.choice()` 调用，选取逻辑从 O(1) 变为 O(1)（纯条件判断）。
